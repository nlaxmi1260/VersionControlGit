Basic Git Commands

  1. git init: Initialize a new Git repository in the current directory
  2. git status: Shows the status of working directory and staging area, displaying changes and untracked files.
    1. untracked: Untracked files are the files which git is not tracking.
    2. staged: Staged Changes means the changes that you have decided to commitâ€¦

      2. git add . : Add all untracked files to the stageing area.
      3. git rm --cached <file-name >: Remove file from staging area to untracked area.
    3. commited: Commit status allows you to mark commits with status information from other systems.
      1. git commit -m "your commit message" : Commit the staged changes to the local repository with a message.
    4. modified: After commited the file you again modified the file .
    5. deleted: The status of a file is deleted either to restore or you have to commit the changes.
  3. git log : Show the commit history for the current branch.
  4. git diff : Show changes between commits ,branches or the working directoy and the index.
  5. git help commit : Help for the commit command.
  6. git checkout <branch-name>: Switch branches or restore working tree files.
  7. git rm <file_name >: To remove a file.
  8. git restore < file_name >: To discard changes in working directory i.e previous commit changes will be avalable.
  9. git restore --staged <file_name >
  10. git rm <file_name >: To remove a file
  11. git rm --cached <file_name >: staged to untracked
  12. git mv < source fileName> < destination filename>: To Rename a file
  13. git commit -a : Whenever a modified file want to commit then we use this command.

Undoing Changes

  1. git reset < CommitId > :Move head to current commitId but content of the file will not change
  2. git reset --hard < CommitId > : Discard all changes and reset.
  3. git revert < CommitId > Create a new Commit that undoes the changes from a previous commit.
  4. git stash Temporarily save changes that aren't ready to be committed, in staged state.
    1. git stash -u :Temporarily save changes that are not ready to be commited in unstaged state
    2. git stash list: Displays list of stashes.
    3. git stash apply : Latest stash will be applied.
    4. git stash apply stash@{n} : This command is used to pick the stash by Id to apply.
    5. git stash pop: This command will apply the latest stash and delete the changes in the Temporarily location.
    6. atleast one commit should be avalable for stash to work.
  5. git rm < file_name >: Remove a file from the working directory and staging area.
  6. git checkout --< file_name >: Discard changes in the working directory for a particular file.

Git Config & Aliases

Personalizing Git configuration can make working with Git faster and more efficient:

  1. Before global configuration state
  2. Empty global configs
  3. git commit --amend --reset-author When user name and user email is not configured in global configs of git, by default email is from the system logged in user and domain is the system name
  4. git branch -m main Initally when defaultbranch is not configured at global level default branch is taken as master. This command is used to update the repository with the global defaultbranch name
  5. git config: Set user-specific configuration values like name, email, and editor.
    1. git config --global user.name "nlaxmi1260" - Setting User Name
    2. git config --global user.email "yaram.nlaxmi@gmail.com" - Setting User Email
    3. git config --global init.defaultBranch "main" - Setting Default Branch
    4. git config --global core.editor "vim" - Setting Default Editor
    5. git config --global --list : to get the global configurations
    6. git config --global --edit : Command to edit the global configs
  6. Git Aliases : You can create aliases for frequently used commands to speed up your workflow.
    1. git config --global alias.co checkout
    2. git config --global alias.cm commit
    3. git config --global alias.st status
    4. git config --global alias.br branch

Collaboration (Remote Repositories)

Managing remote repositories is essential when collaborating with others:

    git remote: Manage remote repository connections.
  1. git remote -v: List remote repositories
  2. git remote add < name > <url >: Add a new remote repository
  3. git remote rm < name >: Remove a remote repository.
  4. git branch --set-upstream-to=origin/main
  5. git clone < repository-url >: Clone a remote repository to your local machine.
  6. git clone -b <branch-name> < repository-url >: Clone a remote repository to your local machine with a specific branch.
  7. git push <remote > <branch >: Push local commits to a remote repository.
  8. git pull <remote > <branch >: Fetch and integrate with another repository or a branch.
  9. git push --set-upstream origin < BranchName >: when you are trying to push to github firsttime

Advance Git Commands

  1. git cherry-pick <commit-hash> Apply a specific commit from another branch into the current branch.
  2. git reflog: View the history of HEAD changes, including commits that may not be in your current branch.
  3. git blame <file_name>: Show the author of each line in a file, helpful for tracking down who made specific changes.
  4. git clean -f: Remove untracked files from the working directory.
